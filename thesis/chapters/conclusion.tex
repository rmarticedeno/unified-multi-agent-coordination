\chapter{Conclusions and Future Work}
\label{chap:conclusion}

The thesis proposes a protocol-adaptable boundary between linguistic proposal and executable multi-agent coordination. RQ1 is answered at the design and prototype level: canonical capabilities, typed constraints, task dependencies, authority, validation and completion contracts, artifacts, attempts, and persisted trace identity must be explicit to justify dispatch. RQ2 is answered architecturally: language interprets and proposes, while deterministic components resolve identity, evaluate enforceable predicates, authorize, refuse, and record. RQ3 is answered through bounded task-local auxiliary specifications whose method, schema, validator, read-only authority, and lifecycle are explicit.

RQ4 is answered as a sequential redesign result within the author-labelled 36-case boundary. Version 0.3 established a negative baseline: the hybrid refused all feasible observations. Version 0.4 constrained model output and deterministically hydrated executable fields; across 540 observations it achieved 90.0\% accuracy, 76.0\% feasible recall, and zero false acceptances. All 15 feasible cases had majority acceptance, but Qwen3 1.7B recall was 28\%, so the pre-registered 60\% per-model floor failed. Separate runtime controls showed zero violations under secure defaults and unsafe decisions when dependency, auxiliary, or trace-evidence controls were removed. No independent adjudication or broad superiority is claimed.

The principal scientific claim is therefore deliberately narrow. The implementation demonstrates how established symbolic, security, persistence, and interoperability mechanisms can be integrated so that unsupported linguistic proposals are refused before action and process evidence survives recovery. It does not prove semantic truth, production safety, malicious-agent resistance, decentralized consensus, Byzantine tolerance, or universal protocol interoperability.

Before submission, the operational work is to raise critical-module coverage, supply institutional and commit metadata, refresh final container digests, and freeze the artifact at one clean tagged commit. Longer-term research should improve small-model draft robustness, reduce repair dependence, add independent corpus adjudication, cryptographic admission, adversarial registries, external domain agents, semantic validators, replanning, and broader deployments.
