\section{Evidence Status}

This chapter distinguishes historical version 0.2 preflight evidence from the completed version 0.3 conformance study. Reports produced by the earlier implementation are legacy artifacts and are excluded from final comparative claims. The 36-case corpus is author-labelled, pre-registered, frozen, and label-hidden during inference; no independent adjudicator was available. The accepted run contains all 540 planned outputs and an identity-complete, deterministically reproducible analysis.

The coverage gate is the principal automated quality blocker. Aggregate coverage is not used to conceal critical gaps: the corrected measurement reports 87\% for the coordination agent, 83\% for feasibility, 77\% for trace validation, 44\% for the PostgreSQL store, 95\% for plan hydration, 89\% for the runtime-ablation runner, 28\% for v0.4 collection, and 24\% for v0.4 analysis. Thus the release remains conditionally ready even though the executed tests pass.

\section{Fail-Closed Symbolic Boundary}

Regression tests show that a promised artifact does not make a validatorless task verifiable; legacy textual constraints become unresolved typed constraints and force refusal; missing JSON-pointer paths, false predicates, canonical capability-ID mismatch, incomplete mode coverage, and nested JSON Schema type violations are rejected. Effective trust is assigned by local admission policy rather than copied from a remote Agent Card. Cards requiring an unavailable credential scheme are refused, and plain HTTP requires an explicit development exception.

Candidate plans remain non-authoritative. Direct delegation is selected only if it is feasible; otherwise bounded candidates are checked and ranked deterministically by task count, auxiliary count, DAG depth, and stable task identifiers. These tests support prototype correctness of the implemented predicates, not mathematical proof of arbitrary plan correctness.

\section{Dependency, Recovery, and Trace Behavior}

Execution tests confirm that a task is dispatched only after every declared prerequisite completes. Failure, timeout, refusal, or unknown outcome skips descendants while independent branches continue. Consumers receive artifacts only from declared dependencies. On takeover, unsafe unknown work is recorded as \texttt{unknown} and is not blindly redispatched. Completion additionally requires the typed completion contract.

Concurrent-session tests show that returned traces contain only persisted events for their session. Authorization precedes attempt start, SDK transport observations are persisted with session, plan, task, and attempt identity, and a trace validator checks ordering and terminal observations. Registration diagnostics remain outside operational session traces.

\section{Corrected End-to-End Harnesses}

The seven deterministic statuses match their references: three successful completions, one authorized runtime failure, and three pre-dispatch infeasibility decisions. The privacy case now derives email and phone values from transcript text; the answers are no longer supplied as separate fixture fields. The task-local auxiliary lifecycle expires after use.

The corrected Docker harness passes eleven checks covering health, registry discovery, direct and decomposed A2A execution, missing capability, insufficient authority, incompatible modes, invalid artifacts, timeout, registry outage, and session replay. The decomposed case originally exposed a stale assumption that every prior artifact should be forwarded. The corrected planner declares dependencies from canonical mode flow, and the harness now verifies that the final consumer receives only its declared predecessor artifact.

The PostgreSQL harness passes five checks: live lease conflict, takeover after an abandoned read-only attempt, process exit after external dispatch of unknown-side-effect work, stale-fence rejection, and database invariant rejection. It records zero duplicate dispatches and repeated effects in these cases, one lease conflict, one stale-write rejection, terminal correctness, and measured recovery latency. These observations do not establish consensus or cover all crash windows.

An additional test launches the vendored official A2A Hello World sample as an independent server, ingests its Agent Card, preserves its \texttt{echo\_bot} skill identifier, authorizes the matching task, and receives its remote artifact. This establishes one concrete interoperability path; it is not evidence of universal A2A compatibility.

\section{Separate Runtime-Only Ablations}

The runtime bundle contains 350 deterministic observations: 100 dependency-dispatch observations, 150 auxiliary-admission observations, and 100 trace-evidence observations. The secure defaults produced zero safety violations across their 150 applicable observations. Removing dependency gating produced 40 unsafe dispatch decisions in the 40 predecessor-missing, failed, timed-out, or unknown observations. Removing auxiliary bounds admitted 40 of 40 deliberately ineligible, persistent, unscoped, or write-authorized specifications; disabling auxiliary behavior admitted none, including ten otherwise valid bounded cases. Suppressing durable diagnostic trace evidence removed the required observation in all 50 trace cases while leaving state-critical persistence unchanged. The observation-file SHA-256 is \texttt{7617bb8da5308b9a\allowbreak 22a7ed7fe12c93d7\allowbreak e0b80484b2a18de2\allowbreak ee44ea3d2e3b7bbf}.

These controls isolate mechanism consequences rather than planning quality. Repetition confirms deterministic behavior of the implemented policy surfaces but does not create 350 independent scenarios: the scenario types, not repeated executions, bound external validity. In particular, the trace result concerns availability of auditable returned evidence, not deletion of leases, fencing tokens, attempts, or terminal results.

\section{Comparative Conformance Study}

The public corpus contains 36 cases and has hash \texttt{b9fd8b6f8398eaf3\allowbreak 3b8676e14288f0d3\allowbreak c4956cca5a9b915c\allowbreak a7b68990167c9c1d}. Its provenance declares author annotation, corpus freeze, and the absence of independent adjudication. The collector verified this exact hash and never included reference labels in prompts. The scorer verified all 540 unique model--seed--case identities before loading labels. Two analysis reruns produced the same SHA-256 digest, \texttt{b70a169d8527b2f4\allowbreak b95a5ef1a7ac083e\allowbreak 0eb81874c896f504\allowbreak 12352b37b14b03a3}. These are leakage, completeness, and reproducibility controls, not evidence that the labels are independently correct.

The hybrid refused every feasible observation: precision, recall, and F1 for feasible acceptance were all zero. Its 58.3\% accuracy therefore reflects correct refusal of infeasible cases rather than useful end-to-end acceptance. On feasible observations, failed predicates most often concerned coverage (225), total assignment (191), completion (185), verifiability (147), and ordering (86). The recurring risks show that local-model proposals omitted accountable executors, canonical requirement identities, promised artifacts, completion contracts, or deterministic validation evidence. The symbolic gate behaved fail-closed, but the linguistic-to-typed-plan bridge did not produce executable plans reliably.

The LLM-only configuration achieved 100\% recall but admitted 303 infeasible repeated observations; its feasible-class precision was 42.6\% and F1 was 59.8\%. The structured oracle achieved 100\% on the author-declared criteria, showing that the symbolic evaluator can accept and refuse correctly when supplied the intended structured request and proposal. At the primary 36-case unit, the oracle differed from the hybrid with Holm-adjusted exact McNemar $p=0.000366$; the LLM-only difference was not significant after correction ($p_{\mathrm{Holm}}=1.0$). Rule-only and the three planning ablations had no discordant decisions relative to the hybrid. This floor effect makes those ablations uninformative rather than evidence that their mechanisms do not matter.

Median collected linguistic latency was 4.84~s overall: 3.61~s for Qwen3 1.7B, 4.66~s for Gemma 4 E2B, and 27.39~s for Qwen3 8B. Configurations share these raw outputs, so this latency is not a between-configuration performance comparison. Symbolic-validation and execution latency belong to the separate controlled runtime evidence.

RQ1--RQ3 receive implementation-level support from the corrected boundary and harnesses. The version 0.3 evaluation answers RQ4 negatively for that protocol: symbolic refusal prevented unsafe acceptance, but model-generated plans made the hybrid unusably conservative. This frozen result supports the authorization boundary while falsifying any claim that the original linguistic integration provided effective accepted coordination. It remains limited to the author's declared criteria on 36 designed cases and does not establish general superiority.

\section{Version 0.4 Linguistic-Bridge Redesign}

Version 0.4 preserves version 0.3 as immutable negative evidence and evaluates a new corpus/protocol hash, \texttt{a0ecd2bf0ebc6740\allowbreak a205108e45385eb1\allowbreak e2e7d7ae00dacab3\allowbreak 8bc123c8ca49aa2d}. The model generated only canonical requirement/capability selections and requirement-level dependencies; deterministic hydration supplied executable fields from the admitted request and registry. One bounded structural repair was attempted in 424 of 540 primary outputs. The complete identity matrix was scored only after its completion sentinel. Two analysis reruns produced the same SHA-256 digest, \texttt{1823095063a91ba2\allowbreak 1df8ff31fd5580b4\allowbreak 51f594e80f511970\allowbreak 32aca7ebdb093c9e}.

The changed corpus wording and bridge protocol also prevent treating the difference from version 0.3 as a direct paired causal estimate.
